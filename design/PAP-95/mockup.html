One conversation language, two surfaces

The conference room and task/issue comments should feel like the same product. Each surface already owns half of the right design, so the proposal cross-pollinates rather than rebuilds.

What changed in rev 4 (your feedback):
  • Status chips follow the PAP-75 guide — in_progress is now blue (liveness), todo amber, done green, in_review violet, blocked red.
  • Conference room: mode removed. No Discuss/Build chip — the room stays a clean attach + send box, so jumping between room and task chat isn't confusing.
  • Approve = standard CTA (white on dark / black on light), no longer blue.
  • Plans stand out from comments — a plan posts as a distinct plan card (document, not a bubble), with Approve / Request changes.
  • Request changes can include screenshots 📎 — shown on the plan card; tracked as PAP-95g.
Blue is liveness

Human messages, sent chats, and in_progress tasks — all blue. Blue = "live right now."

Yellow is planning

A task's planning mode tints its composer yellow — "thinking, not acting yet." (Task only.)

Status is metadata

"Worked 2m", "updated this task", status changes — quiet rows on a rail, never bubbles.

Plans are documents

A plan isn't a comment — it renders as its own card with Approve / Request changes.
